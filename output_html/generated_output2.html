SSLv2 Detected Medium Expand

  • Description:
    Secure Sockets Layer (SSL) version 2 has been detected. This protocol is known to have cryptographic weaknesses as well as other exploitable vulnerabilities.
  • Fix Information: Ensure that applications or services are configured to reject SSL version 2 communications. Note: Most implementations of SSL will provide support for all versions of the protocol. It is up to the application to decide what versions of the protocol it will support and/or refuse. When an SSL connection is negotiated, the client first asks the server application if it will support a certain version of the protocol. This is known as the 'handshake'. If the server responds that the version is supported, then credentials/keys are exchanged to complete the connection. Some server applications, such as IIS, will reject the initial handshake based on its protocol configuration. Other server applications attempt to exchange keys before determining if it will support the requested version of the SSL protocol. Since SSLv2 exploits are normally based on the initial handshake, Retina relies on the response from the handshake request to ascertain a potential vulnerability. Because of this, an application that is configured to reject SSLv2 could still report as vulnerable.
